# 🛒 خادم عروضي

خادم مستقل لمقارنة أسعار البقالة — لا يعتمد على أي طرف ثالث بشكل دائم:
المتاجر تُدار بالكامل من **لوحة التحكم**، وكل متجر مجرد إعدادات JSON تحدد
كيف يُقرأ بحثه وأسعاره (محرك سكرابر عام، بدون كتابة كود لكل متجر).

## نشر الخادم مباشرة (بضغطة واحدة)

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/abukootsh/oroudi)

اضغط الزر، سجّل دخولك بحساب GitHub (لا كلمة مرور جديدة)، ثم Deploy.
بعد اكتمال النشر انسخ الرابط الذي يعطيك إياه Render (شكله `https://oroudi-server-xxxx.onrender.com`)
وحدّث به `PROD_SERVER` في [marker-app/src/api.ts](../marker-app/src/api.ts) ليعمل البحث في النسخة المنشورة على الويب.

## التشغيل محليًا

```bash
cd oroudi-server
npm install
ADMIN_TOKEN=ضع-رمزًا-سريًا npm start
```

- الخادم: `http://localhost:4000`
- لوحة التحكم: `http://localhost:4000/admin` (الرمز الافتراضي `oroudi-admin` — غيّره قبل النشر)
- قاعدة البيانات: ملف `oroudi.db` (SQLite) يُنشأ تلقائيًا مع متجر نموذجي واحد:
  - **التميمي** — سكرابر مباشر على API متجر التميمي الحقيقي؛ أضف بقية المتاجر من اللوحة

## API العام (يستهلكه التطبيق)

| المسار | الوصف |
|---|---|
| `GET /api/search?q=&lang=&city=` | بحث موحّد عبر كل المتاجر الفعّالة، الأرخص أولاً، مع كاش ١٠ دقائق |
| `GET /api/deals` | كل المنتجات المخفّضة من آخر النتائج المخزنة، الأعلى خصمًا أولاً |
| `GET /api/popular` | أكثر ما بحث عنه زوار تطبيقك (آخر ٣٠ يومًا) |
| `GET /api/cities` | قائمة المدن |
| `POST /api/register-device` | تسجيل جهاز لاستقبال الإشعارات الجماعية |

## API الإدارة (ترويسة `x-admin-token` مطلوبة)

| المسار | الوصف |
|---|---|
| `GET /api/admin/stores` | قائمة المتاجر بإعداداتها |
| `POST /api/admin/stores` | إضافة/تعديل متجر |
| `POST /api/admin/stores/:key/toggle` | تفعيل/تعطيل |
| `DELETE /api/admin/stores/:key` | حذف |
| `POST /api/admin/test` | تجربة إعدادات متجر قبل حفظها (يرجع عيّنة نتائج) |
| `GET /api/admin/stats` | إحصائيات البحث والأجهزة وصحة المتاجر |
| `POST /api/admin/notify` | إشعار جماعي لكل الأجهزة عبر Expo Push |

## إضافة متجر جديد من اللوحة — طريقتان

عند «+ إضافة متجر جديد» اختر واحدة:

### 🖱️ HTML (اختيار بالنقر) — للمواقع بلا JSON
1. ضع رابط صفحة البحث في المتجر (استبدل كلمة البحث بـ `{q}`) واضغط «🔎 تحميل الصفحة للاختيار».
2. تظهر الصفحة الحقيقية داخل اللوحة (بلا جافاسكربت الموقع — معاينة آمنة).
3. اضغط «🖱️ حدد بطاقة المنتج» ثم انقر أي منتج — يكتشف النظام العنصر المتكرر تلقائيًا ويعرض عدد المطابقات.
4. اضغط «🖱️ اختر» أمام كل حقل (الاسم، السعر، الصورة…) ثم انقر عليه داخل بطاقة المنتج نفسها.
5. اضغط **🧪 اختبار الإعدادات** للتأكد، ثم احفظ.

### 🔗 JSON (API) — للمواقع التي ترجع بياناتها كـ JSON
1. من متصفحك افتح موقع المتجر، أدوات المطور ← تبويب Network، وابحث عن منتج.
2. اعثر على طلب يرجع JSON فيه النتائج، وانسخ رابطه.
3. ضع الرابط في `search_url` مستبدلًا كلمة البحث بـ `{q}`، وحدّد `results_path` ومسارات `fields`.
4. اضغط **🧪 اختبار الإعدادات** ثم احفظ.

## حالة المتاجر المطلوبة (٨ يوليو ٢٠٢٦)

| المتجر | الحالة | الطريقة |
|---|---|---|
| **التميمي** | ✅ يعمل | JSON — API عام مباشر |
| **الدانوب** | ✅ يعمل | JSON — Algolia (مفتاح بحث عام من كود الموقع) |
| **بن داود** | ✅ يعمل | JSON — نفس منصة الدانوب (Algolia) |
| كارفور | ❌ محجوب | الموقع يرجع صفحة فارغة لأي طلب غير متصفح حقيقي (حماية بوت) |
| نون | ❌ محجوب | يرفض الاتصال من خادم (حماية بوت متقدمة) |
| نينجا (ananinja) | ⏳ يحتاج فحص | موقع React/Next — المحتوى يُبنى بجافاسكربت، لا HTML جاهز ولا JSON ظاهر بعد |
| بنده | ⏳ يحتاج فحص | نفس الحالة |
| لولو | ⏳ يحتاج فحص | الموقع يرجع 403 لطلبات الخادم مباشرة |
| هنقرستيشن | ⏳ يحتاج فحص | Next.js — يحتاج تتبع طلبات API من تطبيق الجوال أو المتصفح |

للمواقع «❌ محجوب» و«⏳ يحتاج فحص»: افتح موقع المتجر من متصفحك على الجوال أو الكمبيوتر، من أدوات المطور (F12) ← تبويب Network، ابحث عن منتج، وابحث عن طلب يرجع JSON — انسخه لي أو ألصقه في اللوحة مباشرة بنمط JSON. أما إن كان الموقع يبني المحتوى في المتصفح فقط (بلا HTML أو JSON واضح)، فالحل يحتاج متصفحًا خفيًا (Playwright) — ميزة قادمة عند الحاجة.

مثال (إعدادات التميمي الفعلية):

```json
{
  "search_url": "https://shop.tamimimarkets.com/api/product?q={q}&limit=100",
  "headers": { "Accept-Language": "{lang}", "User-Agent": "Mozilla/5.0" },
  "results_path": "data.product",
  "fields": {
    "id": "id",
    "name": "name",
    "brand": "brand.name",
    "image": "images.0",
    "url": "slug",
    "unit": "variants.0.name",
    "price": "variants.0.storeSpecificData.0.mrp",
    "discount_amount": "variants.0.storeSpecificData.0.discount",
    "stock": "variants.0.storeSpecificData.0.stock"
  },
  "url_prefix": "https://shop.tamimimarkets.com/product/"
}
```

خيارات إضافية: `original_price` (مسار السعر قبل الخصم)، `in_stock` (قيمة منطقية)،
`price_divisor` (100 إذا كانت الأسعار بالهللات)، `image_prefix`،
`exclude_chains` (لمصادر متعددة المتاجر)، `chain` (مسار اسم المتجر في مصادر مجمّعة).

## متجر بلا JSON؟ نمط HTML

إذا كان الموقع يعرض منتجاته في HTML مباشرة، أضف `"type": "html"` وحدّد محددات CSS:

```json
{
  "type": "html",
  "search_url": "https://store.com/search?q={q}",
  "item_selector": ".product-card",
  "fields": {
    "name":  { "sel": ".title" },
    "price": { "sel": ".price" },
    "original_price": { "sel": ".old-price" },
    "image": { "sel": "img", "attr": "src" },
    "url":   { "sel": "a", "attr": "href" },
    "brand": { "sel": ".brand" },
    "out_of_stock": { "sel": ".sold-out" }
  },
  "url_prefix": "https://store.com"
}
```

- الأسعار تُقرأ من النص تلقائيًا حتى لو كانت «12.95 ر.س» أو بأرقام عربية «١٢٫٩٥»
- كل حقل يقبل `attr` (قراءة خاصية بدل النص) و`regex` (التقاط جزء من النص)
- صفحة تجريبية جاهزة للتعلم والاختبار: `http://localhost:4000/sample-store.html` (الإعدادات كاملة داخل الملف)

> ⚠️ المواقع التي تبني صفحاتها بجافاسكربت في المتصفح (React/Vue) لن يجد فيها HTML جاهزًا —
> هذه تحتاج نمط متصفح خفي (Playwright) — ميزة قادمة، أو ابحث عن API الـ JSON الداخلي للموقع فهو غالبًا موجود.

## التحديث التلقائي (١٢ ليلًا و١٢ ظهرًا)

الخادم يجلب «الكلمات المتتبعة» من كل المتاجر الفعّالة تلقائيًا مرتين يوميًا
(وعند الإقلاع إذا مرّ أكثر من ١٢ ساعة)، فتبقى الأسعار وصفحة «العروض» محدثة حتى بلا زوار.

- عدّل قائمة الكلمات من اللوحة (قسم «🕛 التحديث التلقائي»)
- زر «⟳ تحديث الآن» يجلب كل شيء فورًا
- آخر تحديث ونتيجته يظهران في اللوحة والإحصائيات
